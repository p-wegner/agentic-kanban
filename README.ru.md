# Agentic Kanban

Канбан-доска для управления задачами программирования на базе ИИ. Создана как сфокусированная локальная альтернатива [vibe-kanban](https://github.com/BloopAI/vibe-kanban) — предназначена для однопользовательских рабочих процессов с Claude Code в качестве агента.

Каждая карточка задачи на доске связана с Git-ворктри и активной сессией Claude Code. Основной цикл: **планирование → выполнение (Claude Code) → ревью (diff) → доставка (merge)**.

## Возможности

- **Канбан-доска** — перетаскивание между колонками (Todo, In Progress, In Review, Done, Cancelled), сворачиваемая группа архива
- **Управление задачами** — создание, редактирование, удаление, поиск/фильтрация с подсветкой совпадений, значки приоритета, теги, автонумерация задач
- **Жизненный цикл воркспейса** — создание в один шаг: ветка + Git-ворктри + автозапуск Claude Code. Поддержка прямых воркспейсов (без ворктри) для быстрых задач
- **Вывод агента в реальном времени** — стриминг через WebSocket, чат-подобный ввод с кнопками «Отправить»/«Остановить», поддержка `--resume` для непрерывности сессий
- **Просмотр различий** — единый и разделённый виды с inline-комментариями, статистикой diff, действиями merge и закрытия
- **MCP-сервер** — 10 инструментов для интеграции ИИ-агента (список задач, создание воркспейсов, слияние веток и т.д.)
- **Обновления доски в реальном времени** — WebSocket-push + polling-fallback для изменений между вкладками и через MCP
- **Палитра команд** — Ctrl+K поиск действий с навигацией клавиатурой
- **Мультипроектность** — регистрация нескольких Git-репозиториев и переключение между ними
- **История сессий** — просмотр прошлых сессий агента в каждом воркспейсе без выхода из контекста
- **Обзор ворктри** — просмотр всех Git-ворктри по воркспейсам со статистикой diff и значками статуса

## Стек технологий

| Слой | Технология |
|------|-----------|
| Бэкенд | Hono (Node.js), Drizzle ORM, SQLite |
| Фронтенд | React, TypeScript, Tailwind CSS, Vite |
| Агент | Claude Code через подпроцесс |
| Интеграция | MCP SDK (stdio JSON-RPC) |
| Тестирование | Vitest (unit), Playwright (E2E) |
| Монорепозиторий | pnpm workspaces |

## Начало работы

```bash
pnpm install
pnpm db:setup        # миграции + сидинг + регистрация этого репозитория как проекта
pnpm dev             # запуск сервера (порт 3001) + клиента (порт 5173)
```

Откройте http://localhost:5173 — доска загрузится с 3 активными колонками для зарегистрированного проекта.

Подробные инструкции по установке, включая предварительные требования, см. в [INSTALL.md](INSTALL.md).

### Сброс в чистое состояние

Сначала остановите dev-сервер, затем:

```bash
pnpm db:reset        # очистить БД, выполнить миграции заново, пересоздать теги
pnpm cli -- register .   # перерегистрировать репозиторий
pnpm dev
```

## CLI

```bash
pnpm cli -- register <путь>    # зарегистрировать Git-репозиторий как проект
pnpm cli -- list               # список зарегистрированных проектов
pnpm cli -- unregister <имя>   # удалить проект по имени или ID
pnpm cli -- cleanup            # показать устаревшие ворктри для закрытых воркспейсов
```

## Основной рабочий процесс

1. **Регистрация репозитория** — `pnpm cli -- register /путь/к/репозиторию`
2. **Создание задачи** — добавить задачу на доску через встроенную форму
3. **Запуск воркспейса** — нажать «New Workspace» на карточке задачи (создаёт ветку + ворктри + запускает Claude Code с задачей в качестве промпта)
4. **Ревью изменений** — просмотреть diff в панели воркспейса, добавить inline-комментарии
5. **Слияние** — влить ветку в ветку по умолчанию проекта и закрыть воркспейс

## MCP-сервер

MCP-сервер предоставляет 10 инструментов для интеграции ИИ-агента через stdio JSON-RPC:

| Инструмент | Описание |
|-----------|---------|
| `getContext` | Получить текущий контекст проекта и количество задач |
| `listIssues` | Список задач с опциональным фильтром по статусу |
| `getIssue` | Получить подробную информацию о задаче |
| `createIssue` | Создать новую задачу |
| `updateIssue` | Обновить заголовок, описание, статус или приоритет задачи |
| `listWorkspaces` | Список воркспейсов с опциональным фильтром по задачам |
| `startWorkspace` | Создать воркспейс с Git-ворктри и запустить агента |
| `getWorkspaceDiff` | Получить Git-diff для воркспейса |
| `mergeWorkspace` | Влить ветку воркспейса и закрыть его |
| `closeWorkspace` | Закрыть воркспейс без слияния |

Запуск MCP-сервера:

```bash
pnpm --filter @agentic-kanban/mcp-server dev
```

## Архитектура

```
packages/
├── server/        # Hono API-сервер, SQLite БД, менеджер сессий, CLI
├── client/        # React-фронтенд (Vite + Tailwind)
├── shared/        # Схемы Drizzle, миграции, общие типы
├── mcp-server/    # MCP-сервер (stdio JSON-RPC, 10 инструментов)
└── e2e/           # Playwright сквозные тесты
```

Ключевые паттерны:
- **Серверная агрегация** — сводки по воркспейсам вычисляются в endpoint доски, а не на стороне клиента
- **События доски** — двойной путь: WebSocket-push для мгновенных обновлений + polling-fallback каждые 30с
- **Создание воркспейса в один шаг** — один POST-запрос создаёт запись в БД, Git-ворктри и запускает агента
- **Цепочки возобновления сессий** — внутренний session ID Claude захватывается для `--resume` при перезапуске
